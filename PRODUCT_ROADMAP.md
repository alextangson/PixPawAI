**状态**: 🔴 待修复  
**预估工时**: 15 分钟

**依赖项**: 无依赖  
**成功指标**: 公共 Gallery 只显示 `is_public = true` 的图片  
**创建时间**: 2026-01-17

**描述**:  
当前公共 Gallery 显示了所有用户的图片，包括未选择分享的私密图片，这是严重的隐私泄露问题！

**问题影响**:
- 用户隐私泄露（私密图片被公开）
- 违反用户预期（"私密"实际上是公开的）
- 法律风险（GDPR、隐私保护法）
- 用户信任度严重受损

**技术方案**:
```typescript
// app/[lang]/gallery/page.tsx
// 确保查询只包含公开图片
const { data: images } = await supabase
  .from('generations')
  .select('*')
  .eq('is_public', true) // ← 必须添加这个过滤条件
  .eq('status', 'succeeded')
  .order('created_at', { ascending: false })
```

**验证步骤**:
1. 用户 A 生成图片但不分享
2. 检查公共 Gallery 是否显示该图片
3. 应该只显示用户主动分享的图片

**为什么紧急**:
- 这是数据安全事故，必须立即修复
- 每多存在一分钟，就有更多用户隐私泄露
- 影响品牌信誉和用户信任

**备注**:
修复后需要在所有 Gallery 查询中验证 `is_public` 过滤条件。

### 📂 My Gallery 数据丢失问题（新发现）

**象限**: Q2  
**优先级**: P1 (高 - 数据完整性)  
**状态**: 🔴 待开始  
**预估工时**: 30 分钟  
**负责领域**: 数据库 / 分页  
**依赖项**: 无依赖  
**成功指标**: Dashboard Gallery 显示用户所有历史图片  
**创建时间**: 2026-01-17

**描述**:  
用户 Dashboard 的 "My Gallery" 无法显示所有生成的图片，早期的图片"消失"了。这可能是分页缺失或查询限制问题。

**问题症状**:
- 用户生成 50+ 张图片
- Dashboard 只显示最近的 20-30 张
- 早期图片无法访问（但数据库中应该还在）

**可能原因**:
1. **查询缺少分页** - 只查询前 N 条记录
2. **limit 设置过小** - 例如 `limit(20)`
3. **没有"加载更多"按钮** - 无法查看更多历史
4. **日期过滤错误** - 意外过滤了旧数据

**技术方案**:

**方案 A：无限滚动加载**
```typescript
// 初始加载 20 条
const [offset, setOffset] = useState(0)
const { data } = await supabase
  .from('generations')
  .select('*')
  .eq('user_id', userId)
  .order('created_at', { ascending: false })
  .range(offset, offset + 19)

// 滚动到底部时加载更多
const loadMore = () => setOffset(offset + 20)
```

**方案 B：分页导航**
```typescript
// 显示页码 1, 2, 3...
const page = 1
const perPage = 24
const { data, count } = await supabase
  .from('generations')
  .select('*', { count: 'exact' })
  .eq('user_id', userId)
  .order('created_at', { ascending: false })
  .range((page - 1) * perPage, page * perPage - 1)
```

**验证步骤**:
1. 查询用户的 generations 总数：`SELECT COUNT(*) FROM generations WHERE user_id = ?`
2. 检查 Dashboard 显示的数量是否匹配
3. 如果不匹配，确认是否有 limit 限制

**为什么重要**:
- 用户无法访问自己付费生成的内容
- 数据完整性问题影响信任度
- 可能导致用户投诉和退款请求

**备注**:
优先使用无限滚动，用户体验更好。
