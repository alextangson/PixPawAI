## 📂 文档组织规则

### 何时添加新文档

- **根目录** - 仅限最核心的项目级文档（README、路线图、安全）
- **specs/** - 功能规格、架构设计、API 文档
- **guides/** - 操作手册、配置指南、教程
- **archive/** - 已完成项目的技术记录、历史优化文档

### 文档命名规范

- 使用小写字母和连字符：`my-document.md`
- 英文文档优先，中文文档可用拼音或英文加 `-cn` 后缀
- 描述性命名：`database-migration-guide.md` 而非 `guide1.md`

---

## 🔄 文档维护

- **活跃文档**：定期更新的文档应标注最后更新日期
- **归档时机**：项目完成、功能废弃、文档过时时移至 `archive/`
- **删除标准**：临时文档、已解决问题的清单、一次性任务说明可以删除
